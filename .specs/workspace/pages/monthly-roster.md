# Workspace 月度排班页

## 页面定位

月度排班页负责以高密度矩阵形式展示整月排班，并通过网格内的直接编辑与头部保存动作完成修改，是 Workspace 中交互最复杂的页面之一。

## 源码与依赖

| 类型     | 位置                                                                |
| -------- | ------------------------------------------------------------------- |
| 页面组件 | `src/features/workspace/pages/MonthlyRosterPlannerPage.vue`         |
| 共享组件 | `RosterGrid.vue`、`WorkspacePageHeader.vue` |
| 状态抽象 | `src/features/workspace/composables/useRosterPlanner.js`            |
| 读取接口 | `GET /api/workspace/roster?year=&month=`                            |
| 保存接口 | `POST /api/workspace/roster/save`                                   |

## 页面流程图

```mermaid
flowchart LR
    LOAD[按当前 year/month 加载月视图] --> GRID[渲染团队/人员/日期矩阵]
    GRID --> SELECT[选择单元格]
    SELECT --> BAR[在表格上方使用最小化动作条]
    BAR --> APPLY[点击班次按钮后立即写入本地工作副本]
    APPLY --> SAVE[在页面头部保存修改]
    SAVE --> REFRESH[使用服务端最新月视图回填页面]
    APPLY --> DISCARD[放弃全部待保存修改并恢复最近快照]
```

## 核心交互

- 网格在大数据量下应采用“仅渲染可视区行 + 适度 overscan”的渲染策略，避免一次性挂载整月全部人员行导致首屏和滚动卡顿。
- 选择单元格后不会打开右侧抽屉，而是在表格上方显示最小化的班次动作条。
- 动作条只提供班次按钮与 `Clear`，不重复展示员工、日期、区间提示或额外清除控件。
- 点击班次按钮会立即写入本地工作副本，用户随后通过页面头部的保存动作统一提交修改。
- 页面需要直接在网格中标识未保存改动，避免用户只能依赖页面级提示判断是否有待提交修改。
- 未保存状态不再使用厚重的底部或顶部状态条单独承载，而是折叠进头部的 `Save Changes` 动作中。
- 当存在未保存修改且当前用户可写时，页面只保留轻量的头部保存入口与必要的放弃入口，不再使用遮挡排班矩阵的状态浮层。
- 网格应支持方向键移动当前选中单元格，并允许使用 `Enter` 或 `Space` 进入当前格子的班次动作条操作。
- 网格应在选中任意日期格或键盘聚焦单元格后以清晰可见的 teal 底色高亮该员工整行，冻结的员工姓名列也可点击来选中该员工行；整行高亮只作为定位辅助，不能覆盖当前单元格、拖拽区间或待保存改动的视觉优先级。
- 页面头部应提供“一键复制上月排班”动作：按 staff + 日期把上月排班补齐到本月，仅填充当前为空的格子，不覆盖本月已有内容；若上月日期超出本月天数，则自动忽略。
- “复制上月排班”只应作用于当前用户可编辑的 team；对只读 team 不应生成待保存修改，也不应向无可编辑 team 的用户展示成可用写操作。
- 当存在拖拽区间选择时，区间范围由表格本身的连续高亮直接表达；动作条保持最小化，不额外提供区间提示或单独的清除入口。
- 网格中的 tooltip 等重型悬浮内容应按“当前 hover / focus 单元格”懒挂载，而不是为所有可见班次格子同时挂载完整浮层树，避免大表格滚动卡顿。
- 切换月份时，页面按共享 workspace 月份上下文重新拉取整月视图。
- 若存在未保存修改，切换月份或离开页面前应进行明确确认，降低误丢数据风险。
- 保存成功后用服务端最新响应回填网格，而不是乐观地只改当前单元格。
- 放弃修改会恢复最近一次服务端快照。
- “复制上月排班”属于本地工作副本操作，生成的补齐结果需要像普通编辑一样进入待保存状态，由用户统一保存或放弃。
- 排班网格动效层级为：当前单元格 > 区间选择 > 待保存改动 > 员工行定位 > hover，动效只能强化状态进入，不能让持续动画抢占注意力。
- 状态条、保存成功/失败、导入导出错误使用短暂 Workspace status transition 出现和消失，帮助用户理解操作结果，但不阻塞继续编辑。
- 网格热路径不使用 `transition-all`；单元格仅过渡背景、边框、阴影等低成本属性，班次 chip 只允许轻微 transform。

## 数据边界与约束

- `staffId` 按字符串传输，避免浏览器处理大整数时丢精度。
- `shiftDetailsByTeam` 用于单元格 hover 展示班次说明、时间窗口、时区与跨天信息。
- 单元格展示的班次编码以服务端按 `shiftDefinitionId` 解析后的最新值为准，因此 Shift Code 改名会同步体现在历史排班中。
- `validationWarning` 由页面通过校验中心接口的轻量摘要模式异步补充，不应阻塞月度排班主数据加载，也不应为一句告警搬运完整问题列表；仅当存在 `high` 级别问题时显示主警告。
- 当前 workspace 的 `year`、`month`、`timezone` 应同步到 URL query，提升刷新恢复、回退和分享链接的一致性。
- 页面只负责排班编辑体验，不负责定义班次或维护人员主数据。
- 单元格待保存标记、当前选中状态与员工索引查找应使用可缓存/可索引的数据结构，避免在每个格子渲染时反复线性扫描。

## 维护提示

- 若排班网格继续复杂化，应把网格算法、抽屉协议或保存冲突策略拆成独立子章节。
- 月份上下文、保存载荷、虚拟滚动策略或 tooltip 信息结构变化时，必须同步更新本页与服务端对应 spec。
